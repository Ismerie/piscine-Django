Ex01 : Processus d’affichage d’une page statique.

Cette page décrit le processus entraînant l’affichage d’une page web statique à partir d’un simple template en passant par une vue, de la requête à la réponse.

Processus d'Affichage d'une Page Statique

Le processus d'affichage d'une page statique dans Django implique plusieurs étapes clés :

  1. Définir le template HTML : Le template est un fichier HTML qui sert de base pour la page. Il peut contenir des balises Django pour afficher dynamiquement des données.
  2. Créer une vue : La vue est une fonction Python qui prend une requête web et renvoie une réponse web. Pour une page statique, la vue rend simplement le template HTML.
  3. Configurer les URLs : Les URLs définissent comment les requêtes HTTP sont mappées aux vues. Pour chaque URL, une vue correspondante est définie.
  4. Faire une requête : L'utilisateur fait une requête HTTP en accédant à une URL spécifique.
  5. Traitement de la requête par Django : Django analyse l'URL de la requête, appelle la vue correspondante, rend le template HTML et renvoie la réponse HTTP au navigateur de l'utilisateur.

Exemple Concret

Voici un exemple concret pour illustrer ce processus :

1. Template HTML


{% verbatim %}
<!-- templates/static_page.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Page Statique</title>
</head>
<body>
    <h1>Bienvenue sur la page statique</h1>
    <p>Ceci est une page statique rendue par Django.</p>
</body>
</html>
{% endverbatim %}

2. Vue


{% verbatim %}
# views.py
from django.shortcuts import render

def static_page(request):
    return render(request, 'static_page.html')
{% endverbatim %}

3. URL


{% verbatim %}
# urls.py
from django.urls import path
from .views import static_page

urlpatterns = [
    path('static/', static_page, name='static_page'),
]
{% endverbatim %}

4. Requête et Réponse

Lorsque l'utilisateur accède à l'URL /static/, Django traite la requête, appelle la vue static_page, rend le template static_page.html et renvoie la réponse HTML au navigateur.
